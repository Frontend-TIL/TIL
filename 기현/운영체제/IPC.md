# IPC (Inter Process Communication)

---

## 1. IPC란?

독립적으로 실행되는 프로세스 간에 데이터를 주고받기 위한 통신 방식

* 프로세스 간 데이터 공유 및 협력 수행을 위해 사용
* 운영체제가 제공하는 메커니즘

한 줄 요약
→ 서로 다른 프로세스끼리 데이터를 주고받는 방법

---

## 2. 왜 IPC가 필요한가?

프로세스는 기본적으로 **독립된 메모리 공간**을 가지기 때문

* 하나의 프로세스는 다른 프로세스의 메모리에 직접 접근 불가
* 협력 작업(데이터 공유, 작업 분담)을 위해 통신 필요

예시

* 서버 프로세스 ↔ 클라이언트 프로세스
* 부모 프로세스 ↔ 자식 프로세스

한 줄 요약
→ 프로세스는 서로 격리되어 있어서 협업하려면 IPC가 필요

---

## 3. IPC 방식 종류 (핵심 2가지)

### 1) Shared Memory

여러 프로세스가 **같은 메모리 영역을 공유**

* 데이터를 직접 메모리에 읽고 씀

특징

* 매우 빠름 (복사 과정 없음)
* 동기화 문제 발생 가능 (Race Condition)

---

### 2) Message Passing

프로세스 간 **메시지를 주고받는 방식**

* 운영체제를 통해 데이터 전달

특징

* 안전성 높음 (메모리 격리 유지)
* 상대적으로 느림 (데이터 복사 발생)

---

## 4. Shared Memory vs Message Passing

| 구분     | Shared Memory  | Message Passing |
| ------ | -------------- | --------------- |
| 방식     | 메모리 공유         | 메시지 전달          |
| 속도     | 빠름             | 상대적으로 느림        |
| 안전성    | 낮음 (동기화 필요)    | 높음              |
| 구현 난이도 | 어려움 (동기화 필요)   | 비교적 쉬움          |
| 대표 문제  | Race Condition | 없음 (상대적으로 안전)   |

한 줄 요약

* Shared Memory: 빠르지만 동기화 필요
* Message Passing: 안전하지만 느림

---

## 5. 언제 어떤 방식을 사용하는가?

### Shared Memory 사용 상황

* 성능이 중요한 경우
* 대용량 데이터 처리
* 실시간 처리 시스템

예

* 멀티 프로세스 데이터 처리
* 고속 캐싱 시스템

---

### Message Passing 사용 상황

* 안정성이 중요한 경우
* 프로세스 간 결합도를 낮추고 싶은 경우

예

* 마이크로서비스 간 통신
* 네트워크 기반 시스템

---

## 6. IPC 방식 선택 시 고려 요소

### 1) 성능

* 빠른 데이터 처리 필요 → Shared Memory

### 2) 안정성

* 데이터 충돌 방지 중요 → Message Passing

### 3) 구현 복잡도

* 동기화 처리 가능 여부 고려

### 4) 데이터 크기

* 큰 데이터 → Shared Memory 유리

### 5) 시스템 구조

* 분산 시스템 → Message Passing 적합

---

## 최종 정리

IPC는 독립적인 프로세스 간 협력을 위해 반드시 필요한 통신 방식이며,
성능이 중요하면 Shared Memory, 안정성이 중요하면 Message Passing을 선택한다.
