# Deadlock (교착 상태)

## 1. Deadlock이란?

여러 프로세스가 서로가 가진 자원을 기다리면서 무한히 대기하는 상태

서로 자원을 점유한 채 상대 자원을 기다림
결과적으로 어떤 작업도 진행되지 않음

한 줄 요약
→ 서로 자원을 기다리다가 아무도 실행하지 못하는 상태

## 2. Deadlock 발생 조건 (4가지)

다음 4가지 조건이 모두 만족될 때 Deadlock 발생

1) 상호 배제 (Mutual Exclusion)
자원은 한 번에 하나의 프로세스만 사용 가능
2) 점유 대기 (Hold and Wait)
자원을 가진 상태에서 다른 자원을 기다림
3) 비선점 (No Preemption)
이미 할당된 자원을 강제로 회수할 수 없음
4) 순환 대기 (Circular Wait)
프로세스들이 서로의 자원을 기다리는 순환 구조 형성

한 줄 요약
→ 하나씩 들고 기다리고, 뺏을 수 없고, 서로 물리면 발생

## 3. Deadlock 해결 방법

1) 예방 (Prevention)
- 4가지 조건 중 하나를 제거하여 발생 자체를 방지
- 예: 자원 요청 순서 강제, 선점 허용
- 단점: 자원 활용도 감소, 성능 저하

2) 회피 (Avoidance)
- 시스템이 안전 상태를 유지하도록 요청을 제어
- 대표: 은행원 알고리즘
- 특징
  - 실행 전 안전 여부 판단
  - 위험하면 요청 거부

3) 탐지 (Detection)
- Deadlock 발생을 허용하고 주기적으로 탐지
- 자원 할당 그래프 등을 활용

4) 회복 (Recovery)
- Deadlock 발생 후 해결
- 프로세스 종료
- 자원 강제 회수
- 단점
  - 데이터 손상 가능

5) 무시 (Ostrich Algorithm)
- Deadlock을 무시하고 별도 처리하지 않음
- 이유
  - 발생 확률이 낮고 처리 비용이 큼


## 4. Deadlock vs Starvation

| 구분 | Deadlock   | Starvation     |
| -- | ---------- | -------------- |
| 의미 | 서로 자원을 기다림 | 특정 프로세스만 계속 대기 |
| 원인 | 순환 대기      | 우선순위 문제        |
| 해결 | 구조적 해결 필요  | 스케줄링 개선        |


한 줄 요약
- Deadlock: 서로 기다림
- Starvation: 한쪽만 계속 기다림

## 5. 실제 시스템에서 발생 시 문제

1) 시스템 응답 불가
- 프로세스가 멈추면서 요청 처리 불가능
- 사용자 입장에서 서비스가 멈춘 것처럼 보임

2) 자원 낭비
- 점유된 자원이 반환되지 않음
- 다른 작업도 수행 불가

3) 서비스 전체 지연
- 일부 Deadlock이 전체 시스템으로 확산
- 서버 처리 속도 저하

4) 강제 종료 필요
- 프로세스 종료 또는 재시작 필요
- 트랜잭션 중단으로 데이터 불일치 발생 가능

## 6. Deadlock을 완전히 방지하기 어려운 이유

자원 활용과 성능을 유지하기 위해
일부 위험을 감수하는 것이 더 효율적이기 때문

- 예방 방식은 성능 저하 발생
- 현실적으로는 탐지 및 회복 전략을 함께 사용


## 최종 정리

Deadlock은 여러 프로세스가 자원을 점유한 채 서로를 기다리며
결과적으로 아무 작업도 수행하지 못하는 상태이다.
