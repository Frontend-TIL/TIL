# 이벤트 루프?

## 자바스크립트는 싱글 스레드

싱글 스레드란 한 번에 하나의 일만 수행할 수 있는 것을 말한다.
자바스크립트의 모든 코드들은 앞의 코드가 일을 끝내야 다음 코드가 일을 할 수 있다. 

## 왜? 싱글스레드를 선택했는데?

원래 웹페이지의 보조적인 기능을 수행하려고 만든 언어.
멀티스레드면 무겁고, 어렵고, 동시성 문제도 있다보니, 싱글 스레드 형식이 채택됨.

## 진짜 싱글스레드인거야?

자바스크립트의 메인 스레드인 `이벤트 루프`가 싱글 스레드이기 때문에 싱글 스레드라고 부른다.
하지만 이벤트 루프만 독립적으로 실행되지는 않고 웹 브라우저나 Node js 같은 멀티 스레드 환경에서 실행된다.
-> 즉, 자바스크립트 자체는 싱글 스레드이지만 자바스크립트 런타임은 싱글 스레드가 아니다.

자바스크립트 런타임 자체에서 비동기 API를 지원하는가?
-> X, 자바스크립트 엔진을 구동하는 런타임 환경(브라우저, Node js)가 담당.

## 이벤트 루프란?

싱글 스레드인데 `setTimeout`, `Promise`, `fetch` 같은 **비동기 작업**을 처리할 수 있는 이유가 바로 **이벤트 루프** 때문이다.

이벤트 루프는 콜 스택이 비었는지 확인하고, 대기 중인 비동기 작업의 콜백을 큐에서 꺼내 실행하게 해주는 메커니즘이다.

## 핵심 구성 요소

### 콜스택
- 현재 실행 중인 함수가 쌓이는 공간
- 자바스크립트는 이 스택을 기준으로 코드를 순서대로 실행한다.

### 웹 API / 브라우저 API
- `setTimeout`, DOM 이벤트, `fetch` 같은 비동기 작업을 브라우저가 대신 처리하는 영역
- 자바스크립트 엔진 바깥에서 동작

### 태스크 큐
- 비동기 작업이 끝난 뒤, 실행될 콜백이 대기하는 공간
- 큐에 바로 들어온다고 바로 실행되는 것은 아니고, 콜 스택이 비어야 실행 가능하다.

### 이벤트 루프
- 콜 스택이 비어 있는지 계속 확인
- 비어 있다면 큐에서 작업을 꺼내 콜 스택으로 옮겨 실행

## 동작 흐름

예를 들어 `setTimeout(callback, 0)을 실행하면:
1. `setTimeout`이 호출된다.
2. 콜백이 바로 실행되는 것이 아니라, 브라우저의 웹 API 쪽으로 넘어간다.
3. 지정한 시간이 지나면 콜백이 태스크 큐에 들어간다.
4. 현재 콜 스택의 동기 코드가 모두 끝난다.
5. 콜 스택이 비면 이벤트 루프가 큐에서 콜백을 꺼내 실행한다.

즉, `setTimeout(callback, 0)도 즉시 실행이 아니라,
현재 실행 중인 동기 코드가 모두 끝난 다음 실행된다.


이벤트 루프 덕분에 자바스크립트는:

사용자 입력 처리
화면 렌더링
네트워크 요청 처리
타이머 실행

이런 작업들을 메인 흐름을 막지 않고 처리할 수 있다.

즉, 싱글 스레드인데도 비동기처럼 동작할 수 있게 해주는 핵심 구조다.


## 태스크 큐의 종류

### 1. 매크로 태스크 큐 (Macrotask Queue)

일반적인 비동기 작업들이 들어가는 큐
ex)
- setTimeout
- setInterval
- setImmediate(환경에 따라)
- DOM 이벤트
- I/O 작업

> 이벤트 루프가 한 번 돌때 **매크로태스크 1개** 를 꺼내 실행한다.

### 2. 마이크로 태스크 큐 (Microtask Queue)

우선순위가 더 높은 비동기 작업들이 들어가는 큐
ex)
- `Promise.then`
- `Promise.catch`
- `Promise.finally`
- `queueMicrotask`
- `MutationObserver`

> 현재 실행 중인 코드가 끝난 뒤 매크로태스크를 실행하기 전에 마이크로태스크 큐에 있는 작업을 전부 먼저 처리한다.


```js
console.log("1");

setTimeout(() => {
  console.log("2");
}, 0);

Promise.resolve().then(() => {
  console.log("3");
});

console.log("4");
```

실행결과
```
1
4
3
2
```

풀이
- console.log("1") → 동기
- setTimeout → 매크로태스크 큐로 감
- Promise.then → 마이크로태스크 큐로 감
- console.log("4") → 동기
- 동기 코드 종료 후 마이크로태스크 먼저 실행 → 3
- 그다음 매크로태스크 실행 → 2


> 이벤트 루프는 콜 스택이 비었을 때 태스크 큐의 작업을 가져와 실행함으로써, 
> 자바스크립트가 싱글 스레드임에도 비동기 작업을 처리할 수 있게 해주는 메커니즘입니다. 
> 또한 태스크 큐는 마이크로태스크와 매크로태스크로 나뉘며, 마이크로태스크가 더 높은 우선순위를 가집니다.
