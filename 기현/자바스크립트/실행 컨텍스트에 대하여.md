# 실행 컨텍스트 핵심 정리

## 1. 실행 컨텍스트란?

실행 컨텍스트는 **자바스크립트가 코드를 실행하기 위해 필요한 정보를 관리하는 환경**이다.

쉽게 말하면 아래 2가지를 관리한다.

- **스코프와 식별자 관리**
  - 변수, 함수 이름이 어디에 선언되었는지
  - 현재 어떤 값을 가지는지
- **실행 순서 관리**
  - 지금 어떤 코드가 실행 중인지
  - 함수 호출이 끝나면 어디로 돌아가야 하는지

즉, 실행 컨텍스트를 알면 다음 개념들이 왜 그렇게 동작하는지 이해하기 쉬워진다.

- 스코프
- 호이스팅
- 클로저
- `this`
- 콜 스택

---

## 2. 소스코드의 4가지 타입

자바스크립트는 소스코드를 크게 4가지로 나눈다.

| 타입 | 설명 |
|---|---|
| 전역 코드 | 가장 바깥 영역의 코드 |
| 함수 코드 | 함수 내부 코드 |
| eval 코드 | `eval()`로 실행되는 코드 |
| 모듈 코드 | `import`, `export`를 사용하는 ES 모듈 코드 |

각 코드 타입은 **자신만의 실행 컨텍스트**를 만든다.

---

## 3. 자바스크립트는 코드를 2단계로 처리한다

### 1) 평가 단계
실행 전에 준비하는 단계이다.

- 실행 컨텍스트 생성
- 변수 선언, 함수 선언 먼저 확인
- 식별자를 스코프에 등록

예시:
- `var x` → `undefined`로 초기화
- 함수 선언문 → 함수 객체까지 미리 등록

### 2) 실행 단계
실제 코드가 한 줄씩 실행되는 단계이다.

- 값 할당
- 함수 호출
- 표현식 계산
- 결과를 다시 스코프에 반영

예시:

```js
var x; // 평가 단계: x 등록, undefined 초기화
x = 1; // 실행 단계: x에 1 할당
```

## 4. 실행 컨텍스트가 하는 일
실행컨텍스트의 역할은 크게 2가지.

1. 식별자와 스코프 관리
- 어떤 변수/함수가 유리한지 관리함.
- 이 역할은 내부의 **렉시컬 환경**이 담당.

2. 코드 실행 순서 관리
- 함수 호출과 종료 순서를 관리한다.
- 이 역할은 실행 컨텍스트 스택이 담당.

## 5. 실행 컨텍스트 스택(콜 스택)
자바스크립트 엔진은 실행 컨텍스트 **스택 구조**로 관리한다.

동작 방식은 다음과 같다.

1. 프로그램 시작 시 전역 실행 컨텍스트 생성
2. 함수 호출 시 함수 실행 컨텍스트를 스택 위에 추가
3. 현재 실행 중인 코드는 항상 스택 맨 위 컨텍스트
4. 함수가 끝나면 해당 컨텍스트 제거
5. 이전 실행 위치로 돌아가 실행 계속

예시 흐름

main() → foo() → bar() 순서로 호출되면
```
[전역]
[전역, main]
[전역, main, foo]
[전역, main, foo, bar]
```

bar 종료 후
```
[전역, main, foo]
```

foo 종료 후
```
[전역, main]
```

최종적으로 전역 코드까지 끝나면 스택은 비게 된다.

## 6. 렉시컬 환경이란?
렉시컬 환경은 스코프를 구성하는 내부 구조이다.
-> 쉽게 말하면 다음 정보를 담고 있다.
- 현재 스코프의 변수와 함수
- 각 식별자의 값
- 상위 스코프가 무엇인지

렉시컬 환경은 2가지로 구성

1. 환경 레코드
현재 스코프에 선언된 식별자를 저장한다.
- 변수
- 함수
- 매개변수

2. 외부 렉시컬 환경 참조
현재 스코프의 **상위 스코프**를 가리킨다.

이 참조 덕분에 자바스크립트는 바깥 스코프로 올라가며 변수를 찾을 수 있다.
이것이 바로 **스코프 체인**이다.

## 7. 식별자 검색 과정
코드에서 변수나 함수를 사용하면 자바스크립트 엔진은 아래 순서로 찾는다.

현재 스코프에서 찾는다
없으면 상위 스코프로 이동한다
전역 스코프까지 계속 올라간다
끝까지 없으면 `ReferenceError` 발생

즉, **안쪽에서 바깥쪽으로 검색**한다고 이해하면 된다.


## 8. 전역 코드가 실행될 때

```js
var x = 1;
const y = 2;

function foo() {}
```

평가 단계
- 전역 실행 컨텍스트 생성
- 전역 렉시컬 환경 생성
- var x 등록 → undefined
- function foo 등록 → 함수 객체 생성
- const y 등록 → 초기화 전 상태(TDZ)

실행 단계
- x = 1
- y = 2
- 필요하면 foo() 호출

## 9. 함수 코드가 실행될 때
함수 호출 시 새로운 함수 실행 컨텍스트가 생성된다.

**함수 실행 컨텍스트에 들어가는 것**
- 매개변수
- 지역 변수
- 내부 함수
- arguments
- this

**중요한 점**
>함수의 상위 스코프는 호출된 위치가 아니라, 선언된 위치를 기준으로 결정된다.
>이게 바로 렉시컬 스코프(정적 스코프) 이다.

## 10. 호이스팅과 실행 컨텍스트 관계
호이스팅은 실행 컨텍스트의 **평가 단계**에서 발생한다.

`var`
- 선언과 동시에 `undefined`로 초기화
- 선언 전에도 접근은 가능하지만 값은 `undefined`

함수 선언문
- 선언과 동시에 함수 객체 생성
- 선언 전에 호출 가능

`let`, `const`
- 선언은 미리 되지만 초기화는 실제 선언문까지 미뤄짐
- 초기화 전 구간에서는 접근 불가
- 이 구간을 **TDZ(일시적 사각지대)** 라고 한다.

## 11. 블록 레벨 스코프와 실행 컨텍스트
var
- 함수 레벨 스코프
- {} 블록을 무시하고 함수 단위로 스코프 형성

let, const
- 블록 레벨 스코프
- {} 블록마다 새로운 렉시컬 환경 생성

예:
```js
if (true) {
  let a = 10;
}
```

이 경우 a는 if 블록 안에서만 유효하다.

`for (let i = 0; ...)`

for문에서 let을 사용하면 반복마다 새로운 렉시컬 환경이 만들어진다.
그래서 각 반복의 i 값이 독립적으로 유지된다.

이 개념은 클로저를 이해할 때 중요하다.

---

## 12. 클로저와의 연결

함수가 종료되어 실행 컨텍스트가 스택에서 제거되더라도,
그 함수의 렉시컬 환경을 내부 함수가 참조하고 있으면 메모리에서 바로 사라지지 않는다.

> 즉, 클로저는 실행 컨텍스트와 렉시컬 환경 구조 때문에 가능한 동작이다.


## 핵심

- 자바스크립트는 코드를 평가 → 실행 2단계로 처리한다.
- 실행 컨텍스트는 변수/함수 관리 + 실행 순서 관리를 담당한다.
- 실행 컨텍스트는 스택으로 관리된다.
- 스코프 정보는 렉시컬 환경에 저장된다.
- 변수 검색은 현재 스코프 → 상위 스코프 순서로 진행된다.
- let, const는 블록마다 새로운 스코프를 만든다.
- 함수의 상위 스코프는 선언 위치로 결정된다.
