## 자바스크립트의 핵심, 클로저(Closure) 이해하기

## 1. 클로저(Closure)의 정의

클로저는 자바스크립트 고유의 개념이 아니라, **함수를 일급 객체로 취급하는 함수형 프로그래밍 언어**의 공통적인 특성입니다.

**"클로저는 함수와 그 함수가 선언된 렉시컬 환경(Lexical Environment)과의 조합이다."**

여기서 핵심은 **함수가 선언된 렉시컬 환경**입니다. 즉, 함수가 어디서 호출되었는지가 아니라 **어디에 정의되었는지**에 따라 상위 스코프가 결정된다는 것이 포인트입니다.


**클로저 예시코드**
```javascript
function outerFunction(outerVariable) {
  return function innerFunction(innerVariable) {
    console.log('Outer Variable: ' + outerVariable);
    console.log('Inner Variable: ' + innerVariable);
  };
}

const newFunction = outerFunction('outside');
newFunction('inside');
```
여기서 `innerFunction`은 outerFunction의 내부에 정의되어 있습니다. `innerFunction`은 자신이 생성된 스코프, 즉 `outerFunction`의 스코프를 기억하고, `outerFunction`의 호출이 완료된 이후에도 그 스코프에 접근할 수 있습니다. 그리고 이에 따라 `innerFunction`은 `outerVariable`에도 접근할 수 있습니다. 이것이 클로저가 동작하는 방식입니다.


## 2. 렉시컬 스코프와 [[Environment]]

**렉시컬 스코프 (Lexical Scope)**
- 자바스크립트 엔진은 함수 정의가 평가되는 시점에 상위 스코프를 결정합니다. (정적 스코프)

- **스코프의 실체**: 실행 컨텍스트의 **렉시컬 환경**입니다. 이 환경은 '외부 렉시컬 환경에 대한 참조'를 통해 상위 환경과 연결됩니다(스코프 체인).

[[Environment]] 내부 슬롯
모든 함수는 자신의 내부 슬롯 [[Environment]]에 **자신이 정의된 환경(상위 스코프의 참조)** 을 저장합니다.

함수가 호출되어 실행 컨텍스트가 생성될 때, 이 슬롯에 저장된 값이 바로 '외부 렉시컬 환경에 대한 참조'에 할당됩니다.

## 3. 클로저의 동작 원리

외부 함수보다 중첩 함수가 더 오래 유지되는 경우, 중첩 함수는 이미 종료된 외부 함수의 변수를 여전히 참조할 수 있습니다. 이러한 중첩 함수를 **클로저**라고 부릅니다.

**왜 사라지지 않을까?**
- 외부 함수의 실행 컨텍스트는 실행 컨텍스트 스택에서 제거되지만, **렉시컬 환경까지 소멸하는 것은 아닙니다.**

- 중첩 함수의 [[Environment]]가 외부 함수의 렉시컬 환경을 참조하고 있고, 이 중첩 함수를 누군가(예: 전역 변수) 참조하고 있다면 **가비지 컬렉션(GC)의 대상이 되지 않기 때문**입니다.

**클로저의 조건**
1. 중첩 함수가 상위 스코프의 식별자를 참조하고 있어야 함.

2. 중첩 함수가 외부 함수보다 더 오래 유지되어야 함.

- 자유 변수(Free Variable): 클로저에 의해 참조되는 상위 스코프의 변수.

## 4. 클로저의 활용: 상태 안전 유지
클로저는 상태를 안전하게 **은닉(Information Hiding)** 하고 특정 함수에게만 상태 변경을 허용하기 위해 사용됩니다.

```javaScript
const counter = (function () {
  let num = 0; // 은닉된 변수 (자유 변수)

  return {
    increase() { return ++num; },
    decrease() { return --num; }
  };
}());

console.log(counter.increase()); // 1
console.log(counter.increase()); // 2
console.log(counter.num); // undefined (외부 접근 불가)
```

## 5. 자주 발생하는 실수 (var vs let)
var 키워드로 선언한 변수는 함수 레벨 스코프를 가지므로, 반복문 내에서 클로저를 사용할 때 의도치 않은 결과를 초래할 수 있습니다.

**문제 상황 (var)**
```javaScript
var funcs = [];
for (var i = 0; i < 3; i++) {
  funcs[i] = function () { return i; };
}
// funcs[0]() 실행 시 3 출력 (전역 변수 i를 참조하기 때문)
```

**해결 방법**

1. **즉시 실행 함수(IIFE)** 로 독립적인 스코프를 생성하여 매개변수로 값을 고정합니다.

2. let 키워드 사용: let은 블록 레벨 스코프를 지원하며, 반복마다 새로운 렉시컬 환경을 생성하여 당시의 상태를 스냅샷처럼 유지합니다.

```JavaScript
const funcs = [];
for (let i = 0; i < 3; i++) {
  funcs[i] = () => i;
}
// funcs[0]() -> 0, funcs[1]() -> 1 (정상 동작)
```

>요약
>- 클로저는 태어난 곳(상위 스코프)을 기억하는 함수입니다.
>- 캡슐화를 통해 전역 변수 사용을 억제하고 데이터를 안전하게 보호합니다.
>- 모던 자바스크립트에서는 let, const와 블록 스코프를 통해 클로저로 인한 실수를 방지할 수 있습니다.

## 그래서  언제 클로저 활용하는데?

`클로저는 변수와 함수의 접근 범위를 제어하고 특정 데이터와 상태를 유지하기 위해 자주 활용됩니다. 크게 세 가지 대표적인 사용 사례로 나누어 설명드릴 수 있습니다.`

**첫째, 데이터 은닉에 활용됩니다.** 클로저는 외부에서 접근할 수 없는 비공개 변수와 함수를 만들 수 있습니다. 이를 통해 데이터를 은닉하여 외부 접근을 막고, 데이터 무결성을 유지할 수 있습니다. 예를 들어, 특정 함수 내부에서만 접근 가능한 변수를 생성하고, 이를 조작할 수 있는 함수만 외부로 노출하여 안전하게 데이터를 관리할 수 있습니다.

**둘째, 비동기 작업에 활용됩니다.** 클로저는 비동기 작업에서 이전의 실행 컨텍스트를 유지해야 할 때 유용합니다. 콜백 함수가 비동기적으로 실행될 때 클로저를 사용하면 함수 실행 시점의 변수를 참조할 수 있습니다.

```javascript
function createLogger(name) {
  return function() {
    console.log(`Logger: ${name}`);
  };
}

const logger = createLogger('MyApp');
setTimeout(logger, 1000); // 1초 후에 'Logger: MyApp' 출력
```

위의 예시에서 클로저가 name 변수('MyApp')를 저장하여 1초 후에도 해당 값이 유지되어 출력됩니다.

**셋째, 모듈 패턴을 구현하는 데 활용됩니다.** 모듈 패턴은 특정 기능을 캡슐화하고, 외부에 공개하고자 하는 부분만 선택적으로 노출하여 코드의 응집력을 높이고, 유지보수성을 향상시키는 패턴입니다. 클로저를 활용하면 필요한 함수와 데이터만 외부로 노출함으로써 모듈 패턴을 쉽게 구현할 수 있습니다.
