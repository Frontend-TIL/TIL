# 컨텍스트 관리 마스터하기: AI의 두뇌에 청사진 새겨넣기

## 1. 컨텍스트란?

<img width="1024" height="849" alt="image" src="https://github.com/user-attachments/assets/49f12f03-7577-4764-9a69-b9343378dad3" />

컨텍스트는 AI 모델이 사용자의 쿼리를 해석하고 응답을 생성하는 데 사용하는 정보입니다.
Claude AI Agent <- 여러가지 인풋(지금까지의 대화 내용, Tool을 이용해서 찾아온, 혹은 검색해온 정보들) -> 이 정보들을 기반으로 LLM이 생각하도록 만드는 것

### 컨텍스트의 주요 요소

- 에이전트 시스템에서 성공적으로 작동하는지 여부는 **작업 메모리에 어떤 정보가 포함될지**에 의해 크게 좌우됩니다.
- 대부분의 에이전트 실패는 모델 문제가 아니라, **적절한 컨텍스트 부족** 때문입니다.

### 컨택스트의 구성 요소

- **시스템 프롬프트/지침**: 모델의 행동을 정의하는 기본 지침과 예시, 규칙 등(역할 부여)
- **유저 프롬프트**: 사용자의 즉각적인 요청이나 질문
- **상태/대화 히스토리**: 현재까지의 대화 흐름 및 맥락 정보
- **장기 기억**: 여러 단계를 거친 이전 대화, 사용자 선호도, 과거 프로젝트 요약, 모델이 장기적으로 기억하도록 학습된 정보 모음
- **RAG(검색 기반 증강)**: 외부 문서, 데이터베이스, API 등에서 가져온 최신의 관련성 높은 정보
- **사용 가능한 도구**: 모델이 호출할 수 있는 함수, 내장 툴들의 정의 (예: check_inventory, send_email 등)
- **구조화된 출력**: 모델이 따라야 할 응답 형식 정의(예: JSON)
  - 이 응답은 반드시 다음과 같은 형태여야만 해. {"comment": "추천하는 댓글 내용"}

### 컨텍스트가 중요한 이유

- 실질적으로 **효과적인 AI 에이전트를 만드는 요인**은 복잡한 코드나 모델 품질이 아니라, **얼마나 적절한 컨텍스트를 제공하느냐**입니다.
- 단순한 "데모용" 에이전트는 사용자 요청만 받아들여 기본 응답만 제공하는 반면, "마법 같은" 에이전트는 풍부한 컨텍스트를 고려하여 훨씬 유용하고 인간다운 답변을 생성합니다.
- 비교
  - **저품질(데모) 에이전트**: 단순 요청만 보고 틀에 박힌 답변만 생성
    - 예시: "내일 시간 있으세요?" -> "내일 가능합니다. 몇 시가 좋으신가요?" 와 같이 기계적인 답변
  - **고품질(마법 같은) 에이전트**: 본인의 캘린더, 과거 이메일 히스토리, 상대방 신원 정보, 필요한 도구 호출 옵션 등까지 모두 활용하여 자연스럽고 상황 맞춤형 답변 생성 가능
    - 예시: "내일 시간 있으세요?" -> "내일은 일정이 꽉 찼고, 목요일 오전이 비어 있으니 일정 초대장 보내드렸습니다. 가능하다면 알려주세요."
- 이처럼 알고리즘이나 모델이 중요한게 아니라, 작업에 맞는 올바른 컨텍스트를 제공하는 것이 성공 요인입니다.

## 2. 3가지 유형의 컨텍스트 활용법

AI는 기본적으로 텍스트만 이해하지만, 우리는 이미지, 파일, 문서 등 다양한 형태의 정보를 AI의 '감각'으로 활용할 수 있습니다.

### 2-1. 텍스트 & 코드 컨텍스트: 가장 기본적이고 강력한 '설계도'

가장 일반적인 소통 방식입니다. 하지만 단순히 코드를 복사해서 붙여 넣는 것만으로는 부족합니다.

- **나쁜 예시**
  > (코드 한 덩어리 툭 던져주며) "이거 고쳐줘"
- **좋은 예시**
  > "이 React 컴포넌트의 성능을 최적화하고 싶어. 부모로부터 받는 props가 변경되지 않았을 때 불필요한 리렌더링이 발생하는 게 문제야. React.memo를 사용하는 게 좋을 것 같은데, 어떻게 적용해야 할지 코드로 보여줘. 아래가 현재 코드야"
  
  > (컴포넌트 코드 붙여넣기)
- **Tip**
  - **정밀 타격(`@`)**: `@src/components/Button.tsx`처럼 정확한 파일 경로를 언급하여 AI의 집중력을 높이세요.
  - **가독성**: 프롬프트 안에서도 마크다운(```)을 사용해 코드 블록을 명확히 구분해주면 AI가 훨씬 더 잘 이해합니다.

### 2-2. 파일 & 문서 컨텍스트: AI에게 '참고 도서'를 통째로 주기

때로는 코드 조각이 아니라, 방대한 양의 문서 전체를 AI가 이해해야 할 때가 있습니다.

- **활용 시나리오 ① (API 명세서 분석):**
    
    > [Prompt] "(우리 팀의 20페이지짜리 API 명세서.pdf 파일을 업로드하며) 이 API 명세서를 전부 읽고 이해한 다음, Postman에서 바로 사용할 수 있는 API 테스트 컬렉션 스크립트를 JSON 형식으로 생성해줘."
    > 
- **활용 시나리오 ② (데이터 분석):**
    
    > [Prompt] "(사용자 데이터가 담긴 user_data.csv 파일을 업로드하며) 이 CSV 파일을 분석해서, 국가별 사용자 분포를 계산하고 그 결과를 요약하는 Python Pandas 스크립트를 작성해줘."
    > 
- **활용 시나리오 ③ (여러 파일 동시 참조):**
    
    > [Prompt] "@database/schema.sql 파일과 @server/routes/api.js 파일을 함께 보고, 새로운 '프리미엄 사용자' 등급을 추가하기 위해 두 파일에서 각각 수정해야 할 부분을 모두 알려줘."
    
- **Tip**
  - 위 처럼 정밀 타격을 적용하기 위해 파일을 읽고 지칭하는 것이 너무 부담된다면, 아래 방법을 사용해볼 수 있습니다.
  - Plan Mode 상태에서 "이 작업을 하기 위해서 어떤 파일이 필요할까?" 라는 질문을 하면, 필요한 파일들과 작업 내용을 알려줄 것입니다.
  - "그러면 그 파일들을 기준으로 작업을 진행해" 라는 방식으로 두 가지의 작업을 나눠서 컨텍스트를 준비해주고, 컨텍스트를 같이 전달해 줄 수 있습니다.

### 2-3. 이미지 컨텍스트: 백 마디 말보다 한 장의 '시각 자료'

이미지 컨텍스트는 종종 과소평가되지만, 가장 직관적이고 강력한 무기 중 하나입니다.

- **활용 시나리오 ① (UI 버그 수정):**
    
    > [Prompt] "(모바일에서 버튼 UI가 깨진 스크린샷을 붙여넣으며) 이 이미지처럼 내 React 앱의 버튼 컴포넌트가 모바일 화면에서 깨지고 있어. @components/Button.tsx 파일의 Tailwind CSS 클래스를 수정해서 이 문제를 해결해줘."
    > 
- **활용 시나리오 ② (디자인 시안을 코드로 변환):**
    
    > [Prompt] "(Figma에서 만든 멋진 프로필 카드 디자인 스크린샷을 붙여넣으며) 이 디자인과 똑같은 React 컴포넌트를 만들어줘. 시맨틱한 HTML 태그를 사용하고, 스타일은 Tailwind CSS로 최대한 비슷하게 구현해줘."
    > 
- **활용 시나리오 ③ (다이어그램 이해):**
    
    > [Prompt] "(복잡한 데이터베이스 ERD 다이어그램 스크린샷을 붙여넣으며) 이 DB 스키마를 보고, users 테이블과 posts, comments 테이블을 모두 조인해서, '홍길동'이라는 유저가 작성한 모든 게시물과 그 게시물에 달린 댓글의 개수를 가져오는 PostgreSQL 쿼리를 작성해줘."
    >

## 3. AI의 기억력 붙잡기 - 긴 대화에서 컨텍스트 유지 전략

복잡한 기능을 구현하다 보면 AI와의 대화는 수십, 수백 번을 오가게 됩니다. 이 때 우리는 AI 개발에서 가장 교활하고 위험한 문제에 직면하게 되는데, 이를 **Needle in a Haystack(건초더미에서 바늘 찾기)** 문제라고 부릅니다.

예를 들어, 1번 페이코 작업은 결제 연동 작업이고, 2번 페이코 작업이 UI 작업이라고 했을 때, AI에게 페이코 작업을 해달라고 하면, 컨텍스트 내에서 올바른 값을 찾아내지 못하는 상황에 이르게 됩니다. 즉, 컨텍스트 내에 정보는 많지만 AI에게 혼동을 줄 가능성이 높아집니다.

### 긴 컨텍스트의 문제점

- **정확도 저하**: AI가 엉뚱한 정보(다른 건초)를 '바늘'이라고 착각하고 가져옵니다.
- **핵심 정보 누락**: 중요한 요구사항(바늘)을 아예 발견하지 못하고 무시해버립니다.
- **예측 불가능성**: 이전에 잘 작동하던 명령이 갑자기 이상하게 동작하거나 예상치 못한 결과를 내놓기 시작합니다.
  
-> 긴 컨텍스트는 AI의 가장 강력한 무기이자 가장 치명적인 약점입니다.

### 컨텍스트 관리 전략

#### 전략 1. 주기적인 요약을 통한 컨텍스트 압축

- **개념**: 이 전략은 대화가 길어지면 불필요한 부분을 과감히 삭제하고 **핵심 정보만 모아서 압축된 컨텍스트**를 만드는 전략입니다.
- **비유**: 3시간짜리 회의가 끝난 후, 회의의 핵심 내용을 정리한 회의록과 같다.
- **요약 프롬프트 예시:**
    
    > "좋아, 잠시만. 지금까지 논의한 내용이 너무 길어졌어. 핵심만 요약하고 넘어가자.
    > 
    > 1. **문제점:** 결제 API의 응답 속도가 느리다. (이것이 첫 번째 바늘)
    > 2. **원인:** DB 쿼리의 N+1 문제 때문이다. (이것이 두 번째 바늘)
    > 3. **해결책:** Fetch Join을 사용하기로 최종 결정했다. (이것이 가장 중요한 세 번째 바늘)
    > 
    > 내가 이해한 게 맞지? 맞다면, 이제 이 **요약된 내용(새로운 건초더미)만을 바탕으로** 다음 단계인 `PaymentService.java` 파일의 코드를 수정해줘."
    
#### 전략 2. claude.md와 메모리로 핵심 정보 고정하기

- **개념**: 이 전략은 프로젝트의 가장 중요한 핵심 정보를 고정하여, AI가 다른 정보와 혼동하는 것을 방지하는 전략입니다.
- `claude.md`: 이 파일에 담긴 내용을 프로젝트의 근본적인 규칙이자, 모든 대화의 시작점이 되는 최상위 바늘입니다. AI는 대화가 아무리 길어져도 항상 이 파일의 내용을 최우선으로 참고합니다
- `#`메모리: 대화 중에 새롬게 나온 중요한 정보에 임시 GPS를 다는 기능입니다.
- **방향 재설정 프롬프트 예시:**
    
    > "잠깐만. 지금 우리가 논의하는 내용이 너무 세부적인 기술 구현에만 빠져있는 것 같아. 다시 처음으로 돌아가자. claude.md(GPS가 달린 핵심 바늘)에 정의된 우리 프로젝트의 핵심 목표를 다시 한번 상기해줘. 그리고 현재 우리의 논의가 그 목표 달성에 정말 최선의 방법인지 검토해서 의견을 줘."
    >

이 전략을 통해 AI가 숲속에서 헤매지 않도록, 나침반을 제공할 수 있습니다.

#### 전략 3. 과감한 리셋(`/clear`)

- **개념**: 때로는 컨텍스트가 너무 거대하고 복잡하게 엉켜서, 그 안에서 올바른 정보를 찾는 것 자체가 시간 낭비일 때가 있습니다.
- **비유**: 프론트엔드 UI 컴포넌트 작업을 하다가, 백엔드 데이터베이스 스키마로 컨텍스트가 가득 차버렸다고 가정하고, 이 상태에서 다시 프론트엔드 이야기를 하려고 하면, AI는 백엔드 컨텍스트의 영향을 받아 엉뚱한 답변을 할 확률이 높아집니다.
- **언제 사용해야 할까?** "회원가입 기능 개발"을 끝내고, "CI/CD 파이프라인 구축"이라는 **전혀 다른 성격의 작업을 시작할 때.**
- **왜 필요한가?** 이것은 포기가 아니라, **미묘한 컨텍스트 오염으로 인한 예측 불가능한 버그를 원천 차단하는** 매우 전문적이고 전략적인 결정입니다. `/clear` 명령어로 대화 내용을 깨끗이 지우고, 새로운 주제에 맞는 최소한의 컨텍스트만으로 다시 시작하세요.
