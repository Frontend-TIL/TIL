# 프롬프트 엔지니어링

## 1. 프롬프트 엔지니어링이란?

컨텍스트 관리를 잘하면, AI에게 요리를 하기 위한 최고의 재료를 준비해준 것과 같습니다. 하지만 최고의 재료가 있어도, 레시피가 엉망이면 훌륭한 요리가 나올 수 없습니다. AI에게 최고의 재료로 최고의 요리를 만들어내도록 하는 레시피를 짜는 기술을 '프롬프트 엔지니어링'이라고 합니다.

## 2. 프롬프트의 4요소

### 2-1. 페르소나 (Persona)

AI에게 특정 '역할'이나 '정체성'을 부여하는 순간, AI는 방대한 학습 데이터 속에서 그 역할에 가장 적합한 지식, 말투, 사고방식을 즉시 활성화합니다.

예를 들어, AI에게 단순히 테스트 코드를 맡긴다고 가정해봅시다. 물론 "이 함수 테스트 코드 짜줘."라고 부탁하면 잘 해줍니다. 하지만, 테스트 코드를 작성하는 다양한 방법 속에서 사용자가 원하는 방법으로 하지 않을 수도 있고, 깊이 있는 결과물이 나오지 않습니다.
- **예시**
  - **코드 작성 시**:
  > (Bad) "이 함수 테스트 코드 짜줘."
  > (Good) "너는 TDD(테스트 주도 개발)의 창시자, Kent Beck이야. 이 함수의 모든 엣지 케이스를 고려한 견고한 테스트 코드를 작성해줘."

### 2-2. 목표 (Obejctive)

AI는 사용자가 무엇을 원하는지 잘 모릅니다. 그런 AI에게 "알아서 잘", "적당히"같은 모호한 표현은 마치, 신입사원에게 "알아서 잘 해봐"라고 하는 것과 같습니다. 사용자가 원하는 결과물이 무엇인지, 신입사원에게 업무를 지시하듯, **누가봐도 오해의 소지가 없도록** 명확하고 구체적으로 정의해야 합니다.

모호한 목표를 통한 모호한 결과는 시간 낭비와 불필요한 API 비용을 발생시키며, AI에 대한 신뢰를 떨어뜨리는 가장 큰 원인이 됩니다.

- **예시**
  - **나쁜 예시:**
    
    > "이 코드 리뷰해줘."
    > 
  - **좋은 예시:**
    
    > "아래 JavaScript 코드를 1) 가독성, 2) 성능, 3) 잠재적 버그 세 가지 관점에서 리뷰해줘. 각 항목별로 문제점을 지적하고, 구체적인 개선 방안을 코드 예시와 함께 제시해줘."
    >

### 2-3. 형식 (Format)

AI가 생성한 결과물은 프로젝트에 적용하기 위한 형식과 다를 수 있습니다. 처음부터 내가 원하는 '형식'을 명확히 지정하여, 다시 정리하는 일이 없도록 하는 것이 좋습니다.

- **예시**
  > "data.txt 파일을 읽고 일본 여행지에 대한 정보를 JSON 형식으로 정리해줘. name, category, tags, operatingHours, fee 정보를 포함하도록 해줘."

### 2-4. 제약조건 (Constraints)

제약조건은 AI의 답변이 이상한 길로 빠지지 않도록 도와주는 역할을 합니다.

- **예시**
  > 외부 라이브러리를 사용하지 말고, css만으로 fade-out 애니메이션을 구현해줘.
  
  > 부가 설명 없이 10개의 데이터만 출력해줘.

## 3. 고급 프롬프트 기법

### 3-1. Few-shot 프롬프트 기법

- **개념**: AI에게 업무를 지시하기 전에, 사용자가 원하는 결과물의 형식을 보여주는 예시를 먼저 제공하는 기법입니다.
- **중요한 이유**: AI는 사용자의 마음을 읽을 수 없기 때문에, 단순히 지시사항만 받으면 사용자가 원하는 결과물을 만들기 힘듭니다. 이 때 적절한 예시를 제공하면 AI는 사용자의 의도를 파악할 수 있습니다. 이는 신입 개발자에게 아키텍처를 말로 설명하는 것보다, 잘 작성된 기존 코드 샘플을 보여주는 것이 효과적인 것과 같습니다.
- **예시**
    ```
    당신은 감정 표현을 다양하게 변환하는 데이터 증강 전문가입니다.
    "오늘은 너무 즐거워"와 같은 문장의 의미를 유지하면서 표현을 다양하게 바꾼 새로운 문장 10개를 생성해주세요.

    ## 예시

    ### 원문

    - 오늘은 너무 즐거워

    ### 결과

    - 오늘은 매우 즐겁다
    - 오늘 하루가 너무 행복하다
    - 오늘은 웃음이 가득한 날이다
    - 오늘은 즐거움 그 자체다
    - 오늘은 기쁨의 날이다
    - 오늘은 정말 재미있었다
    - 오늘은 행복감이 가득하다
    - 오늘은 너무 기쁘다
    - 오늘은 즐거운 시간이었다
    - 오늘은 매우 행복했다
    ```

### 3-2. Chain of Thought 프롬프트 기법

- **개념**: 복잡한 문제를 요구할 때, "**단계별로 차근차근 생각해보자.**" 라는 문장을 추가하여, AI가 자신의 생각의 흐름을 스스로 설명하게 만드는 기술입니다.
- **중요한 이유**: LLM은 복잡한 문제에 대해서 성급하게 결론으로 건너뛰려는 경향이 있습니다. 이로 인해, AI의 결과물에서 논리적인 오류가 발생할 가능성이 높아집니다. AI에게 자신의 추론 과정을 풀어쓰는 과정을 추가함으로써 풀어쓰는 과정에서 논리적 오류를 스스로 발견하고 수정하게 됩니다.
- **예시**
  ```
  철수는 사과 5개를 가지고 있었어. 영희가 3개를 주었고, 철수는 사과를 친구에게 2개 주었어. 그 후, 사과를 4개 더 샀어. 철수는 지금 사과가 몇 개인가? 단계별로 생각해보자.
  ```
